## :: تعریف ساختمان داده


>آشنایی با **ساختارهای داده** و **انتخاب ساختار داده بهینه** برای حل مسائل برنامه‌نویسی، یکی از مباحث کلیدی در طراحی سیستم‌ها است. انتخاب مناسب‌ترین ساختار داده می‌تواند تاثیر زیادی بر بهینه‌سازی مصرف منابع سیستم، از جمله مصرف حافظه و زمان اجرا، داشته باشد. به عبارت دیگر، یک ساختار داده باید به گونه‌ای انتخاب شود که حافظه کمتری مصرف کرده و عملیات اجرایی سریع‌تری داشته باشد.

#### پیچیدگی زمانی (Time Complexity) - T(n)

>پیچیدگی زمانی به میزان زمان لازم برای اجرای یک الگوریتم بسته به ورودی‌های آن اشاره دارد. این پیچیدگی معمولاً به صورت تابعی از اندازه ورودی‌ها بیان می‌شود، به عنوان مثال، `O(n)`, `O(log n)`, `O(n^2)` و غیره. هدف این است که الگوریتم‌هایی طراحی شوند که در زمان کمتری برای ورودی‌های بزرگ اجرا شوند.


- **O(1)**: پیچیدگی ثابت (Constant Time)
  - مثال: دسترسی به یک عنصر در آرایه.

- **O(log n)**: پیچیدگی زمانی لگاریتمی (Logarithmic Time)
  - مثال: جستجوی دودویی در یک آرایه مرتب‌شده.

- **O(n)**: پیچیدگی زمانی خطی (Linear Time)
  - مثال: پیمایش یک لیست برای یافتن بیشترین یا کمترین مقدار.

- **O(n log n)**: پیچیدگی زمانی ترکیبی (Linearithmic Time)
  - مثال: الگوریتم‌های مرتب‌سازی سریع (Quick Sort) و ادغامی (Merge Sort).

- **O(n^2)**: پیچیدگی زمانی مربعی (Quadratic Time)
  - مثال: الگوریتم مرتب‌سازی حبابی (Bubble Sort).

- **O(2^n)**: پیچیدگی زمانی نمایی (Exponential Time)
  - مثال: حل مسائل NP-Complete مانند مسئله فروشنده دوره‌گرد (Traveling Salesman Problem) با روش‌های brute force.


#### پیچیدگی حافظه (Space Complexity)

>پیچیدگی حافظه به میزان حافظه‌ای که یک الگوریتم در طول اجرای خود نیاز دارد، اشاره می‌کند. این پیچیدگی نیز بسته به اندازه ورودی‌های داده، متغیر است. به طور کلی، برای بهینه‌سازی پیچیدگی حافظه، باید سعی کرد که حافظه مصرفی کاهش یابد بدون اینکه عملکرد سیستم تحت تاثیر قرار گیرد.


## پیچیدگی زمانی (Time Complexity)

>پیچیدگی زمانی به میزان زمان لازم برای اجرای یک الگوریتم بسته به ورودی‌های آن اشاره دارد. این پیچیدگی معمولاً به صورت تابعی از اندازه ورودی‌ها بیان می‌شود. هدف این است که الگوریتم‌هایی طراحی شوند که در زمان کمتری برای ورودی‌های بزرگ اجرا شوند.


# :: یاداوری

```cpp
int i, n;
i = 1; // (۱ دستور)
```

==نکته== : شرط حلقه محاسبه نمیشود

---

مثال:

```cpp
for (i = 1; i <= 3; i++)
{
    Cout << i;
}
```

## توضیح محاسبه پیچیدگی زمانی حلقه `for`

1. **دستورات بدون حلقه:**  
   - دستور `i = 1` فقط یک بار اجرا می‌شود، پس پیچیدگی آن **1** است.

2. **دستورات داخل حلقه:**  
   - دستورات `Cout << i` و `i++` داخل حلقه سه بار اجرا می‌شوند (به دلیل گردش حلقه).
   - هر بار گردش شامل دو دستور است، بنابراین تعداد کل دستورات داخل حلقه برابر است با
   - \(3 times \* 2 = 6\)
   
3. **مجموع پیچیدگی زمانی:**  
   - با جمع کردن پیچیدگی زمانی دستورات خارج از حلقه (1) و دستورات داخل حلقه (6)، پیچیدگی زمانی کل برابر می‌شود با:

> T(n) = 1 + 6 = 7
> \(T(n) = 7\)

---

### برای دستورات حلقه اینطوری حساب می‌شود:

```cpp
i = 1;
for (; i <= 3;)
{
    Cout << i;
    i++;
}
```
